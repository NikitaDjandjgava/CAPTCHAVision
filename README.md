# CAPTCHAVision: Breaking Text CAPTCHAs with CNNs

> Task 1 of a two-part ML project. This repo covers text-based CAPTCHA recognition using convolutional neural networks.  
> Team: Salome Shioshvili · Jinin Aljayyousi · Nikita Djandjgava · Dana Dabbah

---

## What this does

Takes a 4-character CAPTCHA image like this and automatically predicts the answer.

The target is the **Really Simple CAPTCHA** WordPress plugin with over 1 million active installations. The plugin's author himself says it's no longer secure, which makes it a good study target without ethical concerns.

The pipeline works in three stages:
1. Segment the CAPTCHA into individual character images
2. Train a CNN to recognise each character
3. Use the model to solve new CAPTCHAs

## How to run

### Step 1 — Extract individual letters from CAPTCHAs

```bash
python extract_letters.py
```

Reads every image in `generated_captcha_images/`, segments it into 4 letter crops using OpenCV contour detection, and saves them to `extracted_letter_images/<LETTER>/`. Images where segmentation fails (e.g. too many merged letters) are skipped automatically.

### Step 2 — Train the CNN

```bash
python train_model.py
```

Loads all extracted letter images, trains a CNN on them, and saves the model. Takes a few minutes on CPU.

### Step 3 — Solve CAPTCHAs

```bash
# evaluate on a folder (filename must be the correct answer)
python solve_captchas.py

# solve a single image
python solve_captchas.py path/to/captcha.png
```

## How the segmentation works

Raw CAPTCHA → grayscale → binary threshold (Otsu) → find contours → sort left to right → save each crop.

The only tricky part: sometimes two letters overlap and OpenCV sees them as a single blob. We handle this by checking if a region is much wider than it is tall — if it is, we split it down the middle and treat each half as a separate letter.

## Model architecture

Two architectures are implemented and compared:

**Baseline CNN** (Geitgey 2017)
```
Input (20×20×1)
→ Conv2D(20, 5×5) + ReLU + MaxPool
→ Conv2D(50, 5×5) + ReLU + MaxPool
→ Dense(500) + ReLU
→ Dense(32) + Softmax
```

**Deeper CNN** (proposal architecture 2)
```
Input (20×20×1)
→ Conv2D(32, 3×3) + BatchNorm + ReLU + MaxPool
→ Conv2D(64, 3×3) + BatchNorm + ReLU + MaxPool
→ Conv2D(128, 3×3) + BatchNorm + ReLU
→ Dense(256) + Dropout(0.4)
→ Dense(32) + Softmax
```

Both use Adam optimiser and categorical cross-entropy loss. The character set has 32 classes (A–Z and 0–9, excluding O and I to avoid visual confusion).

## Dataset

- 10,000 CAPTCHA images generated from the WordPress plugin source code
- ~40,000 individual character images after segmentation (~1,000–1,200 per class)
- 80/10/10 train/validation/test split
- No manual labelling — the filename encodes the answer

Sample images from the BenjaminWegener/CaptchaSolver repo were used (MIT licence).

---

## Limitations

- Trained only on images from one CAPTCHA generator — may not generalise to other styles without retraining
- Segmentation assumes exactly 4 characters; will fail on variable-length CAPTCHAs
- The split-in-half fix for merged letters is a heuristic and occasionally wrong
